PROBLEM 5: No files appear in downloads folder
───────────────────────────────────────────────

Symptoms:
[CLIENT 1]> download C:\file.txt
[+] File saved to: downloads/test.txt

But on Linux:
$ ls downloads/
(nothing or error)

Cause:
└─ File transfer failed
└─ Permission issues
└─ File doesn't exist on target

Solutions:
1. Check if file exists on target:
   [CLIENT 1]> shell dir C:\file.txt
   
2. Try a known file:
   [CLIENT 1]> download C:\Windows\System32\notepad.exe
   
3. Check downloads folder on Linux:
   $ ls -la downloads/
