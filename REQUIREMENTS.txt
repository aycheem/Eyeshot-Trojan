NETWORK REQUIREMENTS:
─────────────────────

C2 Server:
- Open port 4444 (TCP)
- Firewall rule: sudo ufw allow 4444/tcp
- Or in Windows: netsh advfirewall firewall add rule name="C2Server" dir=in action=allow protocol=tcp localport=4444

Target Machine:
- Outbound connectivity to C2 server IP:4444
- Network isolation from internet (for lab testing)
- No corporate proxy blocking traffic

═══════════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING INSTALLATION
═══════════════════════════════════════════════════════════════════════════════════

WINDOWS:

Problem: "python: command not found"
Solution:
  1. Reinstall Python from python.org
  2. Enable "Add Python to PATH"
  3. Restart PowerShell/CMD
  4. Verify: python --version

Problem: "pip: command not found"
Solution:
  python -m pip --version
  python -m pip install <package>

Problem: "ModuleNotFoundError: No module named 'psutil'"
Solution:
  pip install psutil --upgrade
  Verify: python -c "import psutil; print('OK')"

Problem: "Permission denied" when running trojan
Solution:
  Run PowerShell as Administrator
  Or check file permissions: icacls trojan_client.py


LINUX:

Problem: "python3: command not found"
Solution:
  sudo apt update
  sudo apt install python3 python3-pip

Problem: "ModuleNotFoundError"
Solution:
  This shouldn't happen - all packages are built-in
  If it does: python3 -c "import <module>" to debug

Problem: "Permission denied" running c2_server.py
Solution:
  chmod +x c2_server.py
  python3 c2_server.py (don't use ./)

Problem: "Address already in use" on port 4444
Solution:
  sudo lsof -i :4444
  sudo kill -9 <PID>
  Or change port in c2_server.py


═══════════════════════════════════════════════════════════════════════════════════
FIREWALL CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════════

LINUX UFW (Recommended):
─────────────────────

# Allow port 4444
sudo ufw allow 4444/tcp

# Verify
sudo ufw status

# To restrict to specific IP:
sudo ufw allow from 192.168.1.50 to any port 4444

# Disable rule
sudo ufw delete allow 4444/tcp


LINUX iptables:
────────────────

# Allow port 4444
sudo iptables -A INPUT -p tcp --dport 4444 -j ACCEPT

# Save rules
sudo iptables-save > /etc/iptables/rules.v4

# View current rules
sudo iptables -L -n


WINDOWS Firewall:
──────────────────

PowerShell (Administrator):
netsh advfirewall firewall add rule name="C2Server" dir=in action=allow protocol=tcp localport=4444

Or GUI:
Settings → Privacy & Security → Windows Defender Firewall → Allow app through firewall

═══════════════════════════════════════════════════════════════════════════════════
UNINSTALL/REMOVAL
═══════════════════════════════════════════════════════════════════════════════════

To completely remove trojan testing system:

WINDOWS:
────────

1. Delete files:
   del C:\trojan_test\trojan_client.py
   del C:\trojan_test\trojan_client.exe

2. Remove persistence:
   reg delete HKCU:\Software\Microsoft\Windows\CurrentVersion\Run /v SystemService

3. Remove scheduled task:
   schtasks /delete /tn SystemUpdate /f

4. Remove packages (optional):
   pip uninstall psutil pillow -y

5. Delete directory:
   rmdir /s C:\trojan_test


LINUX:
──────

1. Stop C2 server:
   killall python3

2. Delete files:
   rm -rf ~/c2_server

3. Remove firewall rule:
   sudo ufw delete allow 4444/tcp

4. Clear logs:
   rm -f ~/.bash_history
   history -c
